A Brief Description

We have created an ATmega32-based game inspired by classic Super Mario Bros. The protagonist "Siam" will journey through the world and try to destroy as many obstacles as possible.

Four 8x8 bicolor LED matrices are used to display the game. Two IR sensors and one sound sensor are used to take input from the player!!!



Required Equipments

  • ATmega32 microcontroller

  • Two IR sensors

  • One sound sensor

  • Four 8x8 bicolor LED matrix

  • Two 4x16 Decoder





How it works?

In this project ATmega32 microcontroller is used for interfacting to various hardware peripherals.

The game is controlled by two IR obstacle sensors and one sound sensor.

One IR senor will shift the game scene to right and another will be used to control jump.

The sound sensor will detect sound above a fixed level and shoot a bullet accordingly.

There will be several obstacles to hinder Siam's journey.

Siam will lose a life whenever he falls through the ground. The game will restart if he has any life left.

A game score(which is proportional to the number of obstacles he passed) will be shown when Siam will lose all the lives.

Algorithm



       1.Start
              2.IF IR.Right is true, the game will move right
              3.IF IR.Jump is true, Siam will take a leap
              4.IF Sound senor is true, Siam will shoot
              5.IF Siam falls through a level, life will decrease
              6.There will be some obstacles to hinder Siam's journey. Siam can shoot them or jump over them using the necessary controls.
                     7.IF life >= 0 , go to step 2
                     8.Game is over
              8.A score will be shown
       16.End

Problems Faced

  1. The project hardwares were costly.

  2. Hardwares are unpredictable.Working circuit was out of order overnight.

  3. Tuning the sound sensor to desired Intensity was a hard nut to crack.

  4. We also had some power issues.Voltage was dropping rapidly.

  5. As we used four LED matrices, the circuit got way too complicated. There were so many wires and it was almost impossible to trace any connection problem .That is why we had to rebuild from scratch to debug.
